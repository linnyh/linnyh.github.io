2020-ACL-Few-shot Slot Tagging with Collapsed Dependency Transfer and Label-enhanced Task-adaptive Projection Network

概要

少样本学习

 目前的深度学习方法都需要大量的标注训练数据，而在真实的落地场景下，大量人工标注数据往往是很难获得的。以任务型对话为例，产品开发时往往难以获得待开发功能的真实用户语料。即使有生语料，任务型对话开发也面临着人工数据标注成本高昂的挑战。同时，像对话系统这样的人工智能应用往往面临着需求频繁变化的问题，导致繁重数据标注任务常常需要反复进行。

 少样本学习（Few-shot Learning）就是专门针对上述问题的新兴研究方向。少样本学习专门研究如何利用过往经验知识，在新的任务上只使用少数几个样例数据进行学习，通常在新的任务上每个类别只有1-5个样例数据。

主要贡献

  • 少样本的序列标注问题中，从原始 domain 学到标签的语义关系，并不能迁移到新的领域（每个领域的标签都不一样）。为解决这个问题，作者提出了 collapsed dependency transfer mechanism，改进了 CRF 中 transition score 的计算
  • 文本之间的相似度难以计算。因为在不同 domain，同样的词语可能有不同的含义。为了解决这个问题，作者提出了 Label-enhanced Task-Adaptive Projection Network，把当前领域 label 的信息也融入样本的特征表示，这样就能分开词语在不同 domain 的特定语义，从而计算 CRF 中的 emission score。

问题定义

 定义输入的句子 $x = (x1,x_2,…,x_n)$, 以及对应的输出标签序列$y=(y_1,y_2,…,y_n)$。一个领域$\mathcal{D} = { (x^{(i)},y^{(i)}) }{i=1}^{ND}$是$(x,y)$的集合。且对于每一个领域都有一个属于自己的标签集合$\mathcal{L}_D = {l_i }{i=1}^{N}$，为了简化，本文假定每一个领域都有$N$个标签。

 少样本模型先在源域${\mathcal{D}1,\mathcal{D}_2,… }$中训练，而后直接应用（不经过微调）于其他未见过的目标领域${\mathcal{D}_1^{‘},\mathcal{D}^{‘}_2,… }$, 每一个目标领域$\mathcal{D}^{‘}_j$都只包含少量的标注样本，被称为support set $\mathcal{S} = { (x^{(i)},y^{(i)}) }{i=1}^{N_S}$。一般support set 是 $N-Way, K-Shot$（N 个 label，每个 label 有 K 个样本）。

 所谓$K-shot$序列标记任务就是给定一个K-shot 的 support set $\mathcal{S}$以及输入序列$x=(x_1,x_2,…,x_n)$, 找到$x$的最佳标注序列$y^*$：

L-TapNet for Emission score

 在少样本学习中，Emission Score利用词（Token）和标签表示（Label Representation）的相似度，得到待预测语句每一个词的标签分布概率分布。这个过程中，标签表示是通过观察少量已有标签样本得到的，如平均所有该标签词的embedding(原型Prototype)，而相似度度量函数是在富数据源于学得的。为了计算它，本文提出了通过改进TapNet（目前最佳的基于少样本学习的图像分类模型）来实现的标签增强的任务自适应投影网络L-TapNet。

2018-EMNLP-Towards Universal Dialogue State Tracking

概要

主要贡献

 本文提出了一种通用对话状态追踪器(Universal Dialogue State Tracking)，该模型的参数量独立于槽值的数量，所有槽共享参数，并使用预先训练好的词向量而不是显式的语义词典以解决下述问题：

  • 模型无法在槽值动态变化的情况下无法工作
  • 模型的参数量与槽的数量成正比
  • 模型基于手工制作的词典提取特征

优点：

  • 模型不需要人工标记的用户话语；
  • 模型对于需要跟踪的插槽可扩展，模型参数的数量不会随着槽数量的增加而增加，因为模型可以在不同的插槽之间共享参数；
  • 模型与槽值的数量无关，这意味着对于给定的槽，只要我们有这个新值的相应字向量，模型就可以对新值进行预测

概览

 $StateNet$只需要以下三部分的数据：（1）用户话语的原始ASR信息（或文字记录)；(2)机器行为的信息；(3）插槽的文字名称和值。用户话语的手动标记不需要作为数据的一部分。$StateNet$在所有插槽之间共享参数，通过它，我们不仅可以在插槽之间传输知识，而且还可以减少参数的数量。

 给定回合级特征向量$is$，我们可以通过LSTM来追踪对话状态。对于当前回合$t$，LSTM 将$i_s$和之前的隐藏状态$q{t-1}$作为输入。得到一个固定长度的预测向量$o_s$，长度为$N_w$：

2-Norm Distance

其中，$\rm{v}_i$是$v_i$的表示向量。如果插槽值$v_i$由多个单词组成，那么$\rm{v}_i$将是所有相应的单词向量的总和。在训练模型时，我们最小化了输出概率与给定标签之间的交叉熵损失。

 由于单词嵌入分布在固定维向量空间上并且持有丰富的语义信息，只要能找到相应的单词嵌入，StateNet就能够跟踪任何新槽或值的对话状态。这就是为什么我们称StateNet为通用对话状态追踪器的原因。

实验

共测试了三种模型：

  • StateNet：不同槽之间没有参数共享，而是分别独立训练，并且参数初始化没有使用预训练模型。优化器使用RMSProp, 学习率 0.0005
  • StateNet_PS：在三个插槽之间进行参数共享。对于一batch中的每个槽，我们使用槽信息和相同的对话信息来推断模型，优化器为RMSProp，学习率 0.0005
  • StateNet_PSI：使用了参数共享，并且但它的参数是用一个预先训练好的模型来初始化的。优化器为Adam，学习率 0.001

模型参数：

$N_c = 128$

$N_w = 300$

$n = 2$

$m = 3$

$c = 4$

结果

如上图，StateNet的三种模型取得了非常好的效果。这是因为参数共享不仅可以防止模型偏离正确的学习过程，而且还可以在不同的插槽之间转移必要的知识。参数初始化为模型提供了在一开始就获得一些基本的语义信息的机会，因为食物槽是最重要和最困难的语义信息。而利用不同槽训练的预训练模型效果如下：

如上图，利用food槽训练过的预训练模型的初始化效果最好。这是因为在联合目标上的良好性能需要一个模型来对所有的插槽做出正确的预测。模型精度最差的一个插槽。最困难的插槽，将极大地限制在联合目标精度度量上的整体模型性能。因此，在最困难的槽上预先训练的模型初始化可以提高模型在其弱点槽上的性能，提高联合目标的准确性，而最容易预测的槽的初始化可能对总体精度没有太大帮助，但反过来会导致槽本身的过拟合问题。
